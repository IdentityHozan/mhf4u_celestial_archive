COSINE

Height of the Building Arch as a Function of Distance

★★★★★
↗ Open in Desmos
"I chose this photo because the smooth curve of the building’s arch was a perfect match for modeling with a trigonometric function. To model the arch appearing in the photo, I began by identifying that its shape closely resembled a cosine wave segment, so I selected the parent function \( y=\cos(x) \) as a starting point. From observing the orientation of the curve in my image, I noticed it peaked at the center rather than the edges, which aligned naturally with the shape of the cosine parent function."
Parent Function\[ y = \cos(x) \]
Transformed Model\[ f(x) = \frac{5}{8}\cos(x - 1.5) + 1 \]

Transformations Applied

The transformation process was a strategic sequence designed to align the theoretical wave with the architectural dimensions of the building:

  • Vertical Compression (\(a = 5/8\)): Initially, determining the vertical scale was challenging because the arch was positioned far from the origin. By temporarily isolating the curve's amplitude in Desmos, I determined a compression factor of \(5/8\) was necessary to match the gentle rise and fall of the physical structure compared to the steeper 1-unit amplitude of the parent function.
  • Horizontal Translation (\(d = 1.5\)): I shifted the graph 1.5 units to the right (\(x - 1.5\)) to align the vertex of the cosine curve with the highest point of the arch. Each unit in this model represents the width of one window pane (approx. 0.8m) to maintain consistent real-world scaling.
  • Vertical Translation (\(c = 1\)): I applied a vertical translation of 1 unit upward (\(+1\)) so the curve’s baseline matched the arch’s height relative to the window base.

Physical Constraints

The domain is strictly restricted to \( D: \{x \in \mathbb{R} \mid 0.25 \le x \le 2.8\} \). I limited the domain to the portion of the arch visible in the photograph, representing the horizontal span measured from the white column base. While the parent cosine function has an infinite domain, the physical arch only exists between these two points; extending the function further would no longer accurately reflect the building's structure.

The parent function \( y=\cos(x) \) has a range of \( R: \{y \in \mathbb{R} \mid -1 \le y \le 1\} \). After applying transformations, the theoretical range of the model becomes \( R: \{y \in \mathbb{R} \mid 0.375 \le y \le 1.625\} \). However, since the domain is restricted to a specific segment of the wave, the practical range is determined by evaluating the function at the domain's boundaries and the vertex.

The range calculation was performed by analyzing the behavior of the function over the interval \([0.25, 2.8]\):

  • Maximum Height: The peak occurs at the vertex \( x=1.5 \), resulting in a height of \( f(1.5)=1.625 \) units.
  • Minimum Height: Since the function is continuous, I evaluated the endpoints at \( x=0.25 \) and \( x=2.8 \), yielding approximate heights of \( 1.197 \) and \( 1.167 \) respectively.
  • Contextual Meaning: These values represent the real-world elevation of the arch measured from the base of the window, where each unit corresponds to a calibrated physical measurement.
Restricted Domain\[ 0.25 \le x \le 2.8 \]
Approximate Range\[ 1.167 \le y \le 1.625 \]

MODEL VI / ABSOLUTE VALUE

Height of the Caduceus V-Shape as a Function of Distance

★★★★★
↗ Open in Desmos
"I chose this photo because it captures our committee's success at the Sunway Medical Science Fair, representing my interest in medical research and contributing to the healthcare landscape in Malaysia. To model the V-shaped base of the Caduceus (medical symbol), I identified the parent function as \( f(x) = |x| \). The transformation process was a strategic sequence designed to align with real-world dimensions:"
Parent Function\[ f(x) = |x| \]
Transformed Model\[ f(x) = 1.3|x - 1.05| + 2 \]

Transformations Applied

  • Horizontal Translation (\(d = 1.05\)): I shifted the function 1.05 units to the right (\(x - 1.05\)) to align the vertex with its position relative to my right shoe. This x-unit was calibrated using the average caution wet floor sign width, which I have labeled as one "Caution: Wet Floor Sign" unit (42.6 cm).
  • Vertical Stretch (\(a = 1.3\)): I applied a vertical stretch factor of 1.3 because the symbol's angle is more acute than the 45° slope of the parent function.
  • Vertical Translation (\(c = 2\)): I translated the graph 2 units upward (\(+2\)) to match the board's elevation. This y-unit was calibrated by my height; each unit represents one "Standard Carry-on Suitcase" (54 cm).

Physical Constraints

The domain is strictly restricted to \( D: \{x \in \mathbb{R} \mid 0.95 \le x \le 1.15\} \). This restriction is contextually justified because the "V" is only a small physical component of the total Caduceus graphic. By limiting the domain to a 0.2 unit span (approx. 8.5 cm in real-world distance), I ensure the model does not extend into space where the function no longer matches the physical object.

The range is \( R: \{y \in \mathbb{R} \mid 2 \le y \le 2.13\} \), determined by evaluating the function at the domain endpoints. This range reflects the actual vertical height of the lines forming the V-shape, beginning exactly 2 vertical units above the floor.
