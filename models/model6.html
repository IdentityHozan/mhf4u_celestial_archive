Transformations Applied

To model the curving pathway accurately against the image, transformations were applied to the parent function in the correct mathematical order:

  • Base Adjustment / Horizontal Stretch (\(b = 1.38\)): First, the base was changed from 2 to 1.38. This alters the rate of growth, creating a more gradual horizontal stretch so that the curvature perfectly matches the incline of the concrete path.
  • Horizontal Translation (\(d = 15.6\)): Next, the graph was shifted 15.6 units to the right to move the main growth curve away from the y-axis, aligning it with the position where the concrete path begins its clear upward sweep.
  • Vertical Translation (\(c = -0.0053\)): Finally, the graph was shifted 0.0053 units downward so that the lower, flatter section of the curve matches the baseline elevation of the path.

Physical Constraints

The domain of this model is restricted to \( D: \{x \in \mathbb{R} \mid 0 \le x \le 20\} \) because the concrete path in the photograph only spans between those horizontal coordinate limits. Beyond \( x = 20 \), the physical path is cut off by the staircase landing platform.

While the parent function has a natural range of \( (0, \infty) \) and a horizontal asymptote along the x-axis, our transformations shift the horizontal asymptote down to \( y = -0.0053 \). In this real-world context, the range changes significantly because it is bounded by our tracking endpoints rather than continuing to infinity. Evaluating the function at the boundaries reveals these physical constraints:

  • Starting Path Elevation: At \( x = 0 \), \( f(0) \approx 0.0013 \), marking where the path enters the left side of the camera frame.
  • Maximum Path Elevation: At \( x = 20 \), \( f(20) \approx 4.1201 \), where the concrete path meets the concrete staircase structure.
