Transformations Applied

The parent logarithmic function \(y=\log(x)\) naturally initiates near the y-axis and increases gradually as x becomes larger. To precisely map this mathematical curve to the contours of the mural artwork, I adjusted the steepness, horizontal placement, and vertical height using the standard mapping form \(a\log(k(x-d))+c\).

  • Vertical Compression (\(a=\frac{7}{20}\)): The value \(a=\frac{7}{20}\), or \(0.35\), compresses the graph vertically. During my initial modeling phases, standard parameters caused the curve to climb far too steeply, deviating from the lines of the wall painting. Utilizing Desmos sliders to systematically adjust values in \(0.05\) increments revealed that a value of \(0.35\) expertly tames the growth rate, forcing the function to visually overlap with the boundaries of the yellow-colored segment and the lines of the painting.
  • Horizontal Compression (\(k=41\)): The value \(k=41\) compresses the logarithmic graph horizontally by a factor of \(\frac{1}{41}\). This adjustment forces the function to experience its initial high-velocity growth over an exceptionally narrow horizontal distance, which perfectly matches the sharper starting section where the artwork first breaks away from the lower-left baseline region.
  • Horizontal Translation (\(d=-0.0122\)): Since the logarithm contains the factored argument \((x+0.0122)\), the graph undergoes a horizontal phase shift of \(0.0122\) units to the left. In the form \(a\log(k(x-d))+c\), this explicitly means \(d=-0.0122\), because \(x+0.0122=x-(-0.0122)\). On this coordinate grid, the x-axis represents horizontal displacement from the left foot base, so this small shift helps align the absolute beginning of the curve with the physical origin of the painted mural edge.
  • Vertical Translation (\(c=3.98\)): The parameter \(c = +3.98\) shifts the entire graph upward. Because the target artwork is suspended high on the upper section of the physical wall rather than resting at floor level, this translation lifts the logarithmic curve to the correct vertical height on the wall painting. On the vertical axis, each graph unit represents approximately \(39.5\text{ cm}\), rounded to about \(40\text{ cm}\), because my height of \(162\text{ cm}\) spans about \(4.1\) vertical units in the photograph. This is close to the width of one small guitar, so the y-axis scale helps connect the graph units to a real-world measurement.

Domain, Range, and Asymptotic Behaviour

The parent function \(y=\log(x)\) is only defined when the interior argument of the logarithm is strictly greater than zero. For this model, the input restriction evaluates as: \[ 41(x+0.0122)>0 \] Since \(41\) is a positive constant, the structural restriction simplifies to: \[ x+0.0122>0 \implies x>-0.0122 \] This mathematically defines the vertical asymptote at line \[ x=-0.0122 \]. The graph cannot touch or pass this line because the logarithm would instantly become undefined, as taking the logarithm of a non-positive real number is impossible.

Vertical Asymptote \[ x=-0.0122 \]
Vertical Shift \[ y=3.98 \]

To ensure the mathematical model behaves realistically within the context of the photograph, a strict contextual domain restriction must be enforced to isolate the specific section of the wall painting: \[ D:\{x\in\mathbb{R}\mid -0.005\le x\le0.5\} \] Without this restriction, the mathematical function would extend infinitely to the right, which fails to represent reality since the physical wall painting terminates. Furthermore, by clipping the lower domain boundary at \(x = -0.005\), the graph is strategically stopped safely away from its vertical asymptote (\(x = -0.0122\)), avoiding erratic downward lines that do not exist on the mural.

Because a transformed logarithmic function with positive \(a\) and \(k\) values is strictly increasing and continuous across the domain interval \([-0.005, 0.5]\), its contextual minimum and maximum heights occur precisely at the domain endpoints.

  • Minimum Curve Height (Left Boundary): Evaluating the function at \(x=-0.005\): \[ f(-0.005) = \frac{7}{20}\log(41(-0.005+0.0122))+3.98 \approx 3.7944 \]
  • Maximum Curve Height (Right Boundary): Evaluating the function at \(x=0.5\): \[ f(0.5) = \frac{7}{20}\log(41(0.5+0.0122))+3.98 \approx 4.4428 \]

Consequently, by tracking the exact vertical displacement from the curve's physical start to its peak within this window, the contextual range is defined with high precision as: \[ R:\{y\in\mathbb{R}\mid 3.7944\le y\le4.4428\} \]

Why This Model Fits and Rejected Alternatives

A logarithmic function was chosen because the wall painting curve increases quickly near its starting point and then gradually flattens as it moves to the right. This behaviour matches the parent function \(y=\log(x)\), which grows rapidly near its vertical asymptote before slowing down. The transformed model uses a horizontal compression to capture the sharp initial rise and a vertical translation to align the curve with the height of the mural. An exponential function was rejected because it would increase more rapidly as \(x\) becomes larger, which is the opposite of the flattening pattern in the image. A linear model was also unsuitable because the mural edge is clearly curved. The logarithmic model fits best because it explains both the rapid initial growth and the gradual leveling of the painting’s edge.
